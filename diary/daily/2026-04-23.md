## Learned

- Rust の module は、C の `#include` のようにファイルを貼り付ける仕組みではなく、crate の中に名前空間と公開範囲を作る仕組みとして読む
- `mod` は module tree に item を登録する宣言で、別の場所から使うときは `crate::`, `super::`, 相対 path などで参照する
- Rust の item はデフォルト private なので、外に見せたい API だけを `pub` で公開する
- `use` は path を短くするために名前をスコープへ持ち込むもので、依存を読み込む操作ではない
- `pub use` は内部の module 構造と、外から見せる API の形を分けるために使える
- `Vec<T>` は同じ型の値を可変長に並べる collection で、`push` するには `mut` が必要
- `Vec<T>` の要素参照を持ったまま `push` できない場合があるのは、内部バッファの再確保で参照先が動く可能性があるため
- `String` は UTF-8 の可変バッファで、byte index と文字の境界が一致しないので単純な添字アクセスはできない
- `HashMap<K, V>` は key から value を引く collection で、`entry(...).or_insert(...)` を使うと「なければ追加、あれば更新」が書きやすい
- collection に所有値を入れると、その値の所有権は collection 側へ移る
