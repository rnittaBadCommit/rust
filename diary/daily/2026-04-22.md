## Learned

- `struct` は C の `struct` に近いが、Rust では `impl` で関連関数やメソッドを型に結び付けられる
- `&self`, `&mut self`, `self` の違いを見ると、そのメソッドが読むだけか、書き換えるか、値を消費するかが分かる
- `Self` は `impl` 対象の型名の別名として読める
- `enum` は C の tagged union に近いが、variant ごとに持つデータを型安全に表せる
- `match` は網羅性をコンパイラが確認するので、処理漏れを減らせる
- `Option<T>` は NULL の代わりに「値があるかないか」を型で表す仕組みとして読むと分かりやすい

## Questions

- `struct` のフィールドを公開するか、メソッド経由にするかの判断基準をもう少し固めたい
- `self` を値で受け取るメソッドを、どんな場面で自然に使うのか例を増やしたい
- `match` と `if let` の使い分けを、実際の演習で確認したい
- C の tagged union から Rust の `enum` に移すとき、どこまで variant に直接データを持たせるのが自然か整理したい
