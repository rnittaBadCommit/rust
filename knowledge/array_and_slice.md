# 配列 `[T; N]` とスライス `[T]` / `&[T]`

Rust では、

- 配列 `[T; N]` は「`T` を `N` 個ちょうど持つ値」
- スライス `[T]` は「`T` が連続して並んだ領域そのものを表す型」
- 実際によく使うのは `&[T]` で、「その連続領域への借用」

です。

## まず結論

```rust
fn main() {
    let a: [i32; 4] = [10, 20, 30, 40];

    let whole: &[i32] = &a;
    let part: &[i32] = &a[1..3];

    println!("{a:?}");
    println!("{whole:?}");
    println!("{part:?}");
}
```

- `a` は配列。値を 4 個そのまま持つ
- `whole` は配列全体を見るスライス
- `part` は一部だけを見るスライス

`part` は新しい配列を作っていません。
元の `a` の一部を「ここからここまで」と参照しているだけです。

## C でたとえると

```c
int sum(const int *xs, size_t len);
```

Rust の

```rust
fn sum(xs: &[i32]) -> i32
```

は、感覚としてかなりこれに近いです。

- `const int *xs` に近いもの: 先頭要素への参照
- `size_t len` に近いもの: 長さ

つまり `&[i32]` は「ポインタ + 長さ」に近い見方でほぼ合っています。

## 何が違うのか

### 配列 `[T; N]`

- 値を所有する
- 長さ `N` は型の一部
- `[i32; 4]` と `[i32; 5]` は別の型

```rust
let a: [i32; 4] = [1, 2, 3, 4];
```

これは「`i32` を 4 個持つ値」です。
長さ 4 という情報がコンパイル時に確定しています。

### スライス `[T]`

- 所有しない
- 長さは実行時に決まる
- 単体の `[T]` はサイズ不定なので、普通は値としてそのまま持たない
- ふつうは `&[T]` や `&mut [T]` として使う

```rust
let a = [1, 2, 3, 4];
let s: &[i32] = &a[1..3];
```

`s` は `[2, 3]` というコピーを持っているわけではなく、
`a` の中の一部分を指しています。

## 「スライスはオーバーロードできる？」への答え

基本的には違います。

スライスは Rust に最初からある型です。
「オーバーロードできる型」ではなく、「標準で定義されている連続領域ビュー」です。

混乱しやすいのは `a[1..3]` の見た目です。
これは「範囲指定で一部を取り出す」文法に見えますが、
内部では添字演算に関する仕組みが使われています。

ただし、学習の最初の段階では、

- `a[1..3]` は「配列や `Vec` の一部をスライスとして見る」

と理解しておけば十分です。

## API で `&[T]` がよく使われる理由

```rust
fn sum(xs: &[i32]) -> i32 {
    let mut total = 0;
    for x in xs {
        total += *x;
    }
    total
}

fn main() {
    let a = [1, 2, 3, 4];
    let v = vec![1, 2, 3, 4];

    println!("{}", sum(&a));
    println!("{}", sum(&v));
    println!("{}", sum(&v[1..3]));
}
```

`&[T]` を受け取る関数にしておくと、

- 配列全体
- `Vec` 全体
- 一部分

を同じ形で受け取れます。

`[T; N]` を引数にすると、長さごとに別の型になってしまい、使いにくいです。

## 一言でまとめると

- 配列 `[T; N]`: データ本体を持つ固定長の箱
- スライス `&[T]`: その箱や `Vec` の中の連続した範囲を借りて見る窓
